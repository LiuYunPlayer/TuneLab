# 钉死音素「时长(duration) 模型」 — 手测用例

只覆盖本次需求受影响范围：**钉死音素的存储 / 定位 / 伸缩 / 越界引导 / 延音铺设 / 跨 note 压缩 / 推挤编辑 / 旧工程兼容**。
不重复已通过的基线（会话调度、和弦重叠创建、属性面板等各有独立文档）；voice 去重叠的单声部 + 暗色显示见 `VOICE-NOTE-OVERLAP-TEST-CASES.md`。

模型要点（供判定期望时对照）：
- 每个音素存 **`{时长 Duration, 权重 Weight}`**，**位置不存、由布局派生**：辅音(`w=0`)用固定时长；元音(`w>0`)时长为**填充派生**（布局忽略其记录值）。
- **布局（自然位置）**：**首元音(w>0)起点对齐 note 起点**（歌唱对拍）；**前辅音从元音起点往左依次累积各自时长**（可任意加长、向 note 前自由越界）；**元音 + 后辅音往右**——辅音用固定时长，**元音填充剩余空间到 note 满末**（含 melisma 铺过乘客）、多元音按权重分摊。故元音吸收 note 伸缩；末音素恒落在满末。
- 权重在**锁定那一刻**取合成产物的伸缩权重（前/后辅音 `w=0` 固定、元音 `w=1` 可伸）。典型音节 `[前辅音(w0), 元音(w1), 后辅音(w0)]`。
- **跨 note 去重叠（全局两阶压缩）**：相邻钉死 note 靠近重叠时，由**全局布局算法**（`PhonemeLayout`，跨 note）统一求解。每个 note 边界的吸收跨度 = `[前 note 末元音 … 后 note 首元音]`：
  - **① 元音先让**（w>0，最多到 0）；**② 元音耗尽仍超 → 辅音簇（w=0，前 note 尾辅音 ∪ 后 note 前辅音）按标称长度等比压**（V1 无最小地板、可压到 0）。
  - **钢琴窗钉死显示**（连续钉死 note 的 3-note 窗口）用宿主布局算法；**合成**由插件自带参考布局（照抄宿主算法、见 V1.Voice 的 `RefLayout`，钉死 note 用其几何覆盖预测）。两者语义一致，故合成产物与显示对得上。
- **推挤编辑**：拖某音素边界 = 改该音素**时长**；相邻音素时长不变、**整体平移（推挤）**而非互相挤占长度，元音(填充)吸收变化。**前辅音可自由向前伸展**（加长其时长 → 头向 note 前延伸，不被任何锚固定）。**显示位经压缩**故拖拽**反解跟手**：压缩区里边界被布局钉住时，跟到钉点即止。
- **乘客 / 延音符** = 歌词 `-`（**唯一**延音符记号）**且**无归属自己的音素。合成后以 origin（有无自己音素）为权威；**合成前**邻居还没音素时用 `-` 约定预测乘客 —— 真歌词 note **不**被铺过。**空歌词不是延音符**：插件按"不认识的歌词"返回单个 `""` 无效音素。乘客在音素时间线上透明、被前一音节铺过。
- **末音素的尾不可拖**（派生量）；**核起点（=note 起点）不可拖**（核恒在音符头，是固定拍点）。要让元音发声更长 → 把 note 画长（元音填充随满末伸缩）。前辅音的**负向越界**（note 之前）天然由其时长决定、可拖加长。

## 前置准备

1. **拿新二进制**：先**关掉 TuneLab**，再跑 App（自动 build 最新宿主）。插件（v1-voice）本轮已改并**自动装好**，无需手动安装。声库 UI 显示名 **「Alice (V1 Test)」**。
2. 新建空工程 + 一条 MidiPart 挂「Alice (V1 Test)」。
   - **音素按歌词解析**（元音字母 = a/e/i/o/u）：切「前置辅音簇 + 元音簇 + 后辅音簇」。前置辅音 `IsLead=true`、刚性 `w=0`、从核起点往左累积；元音 `w=1` 填充；后辅音 `w=0` 占组末。
     例：**`a`** 纯元音；**`ka`** k+a；**`kas`** k/a/s；**`skat`** s,k(**多前置**)/a/t；**`kalt`** k/a/l,t(**多后辅音**)。

4. **锁定音素**：在音素带上**拖动某 note 的任一音素起边界**即把**该 note**（仅它自己，不再连带前后 note）锁为钉死。未锁定时显示合成产物（合成音素显示生命周期另见 `PHONEME-DISPLAY-TEST-CASES.md`）。

## 用例 1 — 合成显示：引导辅音越界 + 元音

画两个**紧邻**的歌词 note A、B（如各 1 拍，歌词 **"ka"、"ki"**），合成完成。

- **期望**：音素带每个音节显示 `k`（引导辅音）+ 元音；B 的 `k` **起点落在 A 的尾巴内**（越界到前一 note），即 A 的元音在 A 尾前 0.1s 处结束、紧接 B 的 `k`。
- 判定：引导辅音可越界到上一 note（首边界 offset 为负）；元音末被后盖前封顶到 B 的 `k` 起点。

## 用例 2 — 延音铺设（乘客 melisma）

画一个歌词 note A（歌词 **"ka"**），后面**紧邻**两个延音符 note（歌词 `-`，可拆音符或直接输入 `-`），合成。

- **期望**：A 的元音**铺满 A + 两个延音符**整段（一直到最后一个 `-` 的尾）；两个 `-` note **不显示自己的歌词框/音素**（它们是乘客、透明）。
- 对照（错误结果）：元音只到 A 自己的尾、`-` note 各画一个音素 → 说明乘客铺设失效。

## 用例 3 — 锁定后拖伸 note：元音跟随、辅音固定

合成用例 1 的 A，**拖动 A 的某音素起边界**锁定它。再拖 A 的**右边界**把 A 拉长 / 压短。

- **期望**：引导辅音 `k`、后辅音时长**固定不变**（刚性）；**元音吸收长度变化**（拉长时元音变长、压短时变短），末音素恒随 note 满末跟随。
- 判定：辅音固定时长，元音填充随 note 伸缩。

## 用例 4 —（改）末音素尾派生、不可单独拖

锁定一个**孤立**歌词 note A（其后无 note、无乘客）。

- **期望**：A 的元音**恰好填满到 A 尾**（own 尾）；音素带上**末尾没有可拖的边界柄**（只有 n 个起边界柄）。拖伸 A → 元音末随 A 尾跟随；平移 A → 整体随之平移。
- 判定：末音素的尾是派生量（own 尾 + 乘客 + 后盖前），不存、不可越界拖出 note 外。要更长的发声请把 note 画长。

## 用例 5 —（关键）锁定中间 note 移到空旷处，元音收回填满

画三个**紧邻**歌词 note A、B、C（如 **"ka"、"ki"、"ku"**）。合成后注意：受 B 的引导辅音挤压，**A 的元音在 A 尾前 0.1s 处就结束**（B 的 `k` 占了 A 最后 0.1s）。

1. **拖 A 的音素起边界锁定 A**。此时 A 的元音末仍在 A 尾前 0.1s（被 B 的 `k` 盖着）。
2. 把 **A 整体平移到右侧空旷无 note 处**。
3. **期望**：A 的元音**收回那 0.1s、填满到 A 尾**（无后续 note 再挤压、无乘客）；引导辅音 `k` 仍在 A 头前 0.1s。
- 对照（错误结果）：A 的元音末仍停在"距尾 0.1s"处、留固定缺口 → 说明末尾被错误烘焙、未派生。

## 用例 6 — 后盖前 / 重叠

> 注意：voice 的 note 经宿主去重叠后单声部（见 `VOICE-NOTE-OVERLAP`），重叠尾被钳。此处验证音素层的后盖前：

画 A、B 两 note 紧邻或轻微交叠。合成。

- **期望**：A 的元音正向铺设止于 B 的首边界（B 的 `k` 起点），即"后头盖前尾"；紧邻 / 交叠表现一致，无错位 / 反相。

## 用例 7 — 间隙断开（有空隙 = 两 note 音素互不影响）

画 A、（**留一段空隙**）、B 两个歌词 note。合成。

- **期望（空隙够大）**：A 的元音**到 A 尾即止、不铺进空隙**（空隙无乘客 note、是静音）；B 的前辅音落在空隙里、从自己的引导辅音开始，与 A 不交互。
- 判定：乘客铺设只跨"有 note 的乘客"，空隙处无 note 故不铺。
- **有空隙时音素互不影响（关键·显示侧）**：把 B 改成前辅音较长的（`skat`，或拖长 B 前辅音）使其自然长度超过空隙，把 B 慢慢往左推近 A（仍留空隙）。
  - **期望**：B 的前辅音**保持自然长度、可自然探入空隙并在显示上与 A 重叠**，但 **A 的元音不被推挤 / 截短、B 的前辅音也不被压缩**——两 note 音素互不影响。B 在空隙内移动时**固定音素不跳变**。
  - 对照（错误结果 / 已废弃）：B 前辅音随空隙压到 0、或推挤截短 A 元音 → 说明又在空隙处做了跨 note 协同。
- **拉到相接才协同**：把 B 拖到与 A **相接 / 重叠**（无空隙），此时 B 的前辅音**越界进 A 元音尾、元音先让**（用例 1 协同发音、可拖拽调）。即"想要可调就把音符拉来相接"。
- **合成侧由引擎决定**：引擎对微小空隙可合并连续推理（→推挤）或分片独立产出（→前辅音与 A 尾在时间上重叠混音，"重叠发声"效果）。合成产物与显示预览不符时仅音素线与波形错位、非致命。

## 用例 8 —（关键·推挤）拖辅音边界推开邻居、不挤占；前辅音自由伸展；跟手

锁定一个有**多个辅音**的 note（如前面再补辅音，或用 `kat` 观察前辅音 k / 后辅音 t）。

- **拖前辅音 `k` 的起点（最左柄）向左**：`k` **变长、向 note 前自由伸展**（无固定上限）；其后音素不动。拖回则变短。
- **拖后辅音 `t` 的起点**：`t` **变长、向左吃进元音**（元音缩短吸收）；`t` 之后的音素（若有）**保持各自长度、整体平移**，不被挤占。
- **跟手**：拖动时边界**跟随鼠标**；若该 note 紧邻另一钉死 note 被压缩、边界被布局钉住，则跟到钉点即止（不再乱跳 / 跑飞）。
- **元音起点（=note 起点）**不可拖（对拍锚）。
- 对照（错误结果）：拖中间辅音把**旁边辅音挤短**（而非推开）→ 说明退回了边界独立模型；或拖一点边界**跑很远 / 不跟手** → 说明压缩反解失效。

## 用例 9 — 旧工程载入兼容（startTime/endTime → duration + IsLead + weight）

用**改动前**版本存的旧 `.tlp` / `.tlp` cbor（含钉死音素）；用本版本打开。旧音素以**相对音符头的秒** `startTime/endTime`（音符头 = 0，前置辅音 `startTime` 为负）存储，无 `duration/weight/isLead` 字段。本版本一次性转换：

- **时长**：`Duration = max(0, endTime − startTime)`。
- **前置标记 `IsLead`**：按**区间中点落在音符头之前**判定——`(startTime + endTime) / 2 < 0` ⇒ `IsLead = true`（旧版引导辅音多半 `startTime` 负、整段在音符头前，故被正确识别为前置）。
- **权重 `weight`**：旧模型无弹性概念，**第一个非前置音素默认为元音**（`weight = 1`、弹性、吸收 note 伸缩 / 压缩），其余（前置辅音 + 后辅音）刚性 `weight = 0`。这与布局「单一弹性核」的假设一致，保证核起点（左右锚）不被压缩推动。
- **期望**：旧音素载入不报错；引导辅音落在音符头前、其余音素从音符头往右、随 note 伸缩。再次保存、重开能正常往返（新版写 `duration/weight/isLead`，不再写 `startTime/endTime`；旧的 `anchor/offset/leadOffset` 被丢弃）。
- 若手头无旧工程：本版本内锁定音素后存盘、重开，音素应原样还原（往返一致）。

## 用例 10 — 合成前真音节不被越铺（无"先越界后缩回"）

锁定一个歌词 note A，使其有钉死音素。在 A **紧邻其后**新画一个**有真歌词**的 note B（如 **"ki"**），**先别合成**。

- **期望**：A 的元音末**停在 B 的起点**、不越进 B（B 虽暂无音素，但歌词非 `-` → 不被当乘客铺过）。合成后 A 的元音末落到 B 的引导辅音前（仅左移约 0.1s），**不出现"合成前越进 B、合成后猛缩回"的跳动**。
- 对照（错误结果）：合成前 A 的元音整段盖进 B、合成后才缩回 → 说明乘客判据漏了"非 `-` 不铺过"。
- 对照（延音符仍铺）：把 B 的歌词改成 `-`，合成前 A 的元音应**铺过 B**（B 是延音符乘客）。

## 用例 11 — 空 / 不认识歌词 = 无效音素

把某 note 歌词清空（或填一个声库不认识的记号），合成。

- **期望**：该 note 产**单个 `""` 无效音素**（音素带该段为空符号），它**有自己的音素**故不被前一音节当乘客铺过；前一音节的元音止于该 note 起点。
- 判定：空歌词**不**等同延音符 `-`；唯一延音符是 `-`。

## 用例 12 —（关键·后辅音）固定长度 + 推挤（元音先让）

画歌词 note A = **`kat`**（前辅音 k + 元音 a + 后辅音 t），合成。**拖 A 的某音素起边界锁定**。

- **期望（基本）**：音素带显示 `k`（头前越界、刚性）+ `a`（弹性）+ `t`（音节末固定一小段、刚性）。
- **拖伸 A**：元音 `a` 吸收长度变化；前辅音 `k`、后辅音 `t` **长度基本不变**（k 钉在头、t 钉在尾）。
- **推挤（中度重叠）**：在 A **紧邻其后**画 B = `kat` 并合成。B 的前辅音 `k` 挤压 A 的尾 → **A 的后辅音 `t` 长度不变、整体左移**，被挤短的是**元音 `a`**（元音先让、未耗尽）。
- 对照（错误结果）：被挤短的是后辅音 `t`（t 贴着 B 的 k 缩短、长度变小）→ 说明压缩没按两阶分级（元音应先让）。
- **极端重叠**：A、B 靠得很近（A 短、重叠量超过元音 `a` 的长度）→ 元音 `a` **压到 0**，再 `t`+B 的 `k` 簇**一起等比压**（见用例 13）。

## 用例 13 —（关键·跨 note 辅音簇压缩，头号修复）

> 这是本轮全局布局的核心修复：旧的 per-note 封顶表达不了**跨 note** 的辅音簇压缩，后 note 的前辅音会直接叠到前 note 的尾辅音上。

画两个**靠近的钉死** note：A = **`kas`**、B = **`bus`**（各自**锁定**音素，使其为钉死 note）。把 B 往左拖近 A，直到 A 的尾辅音 `s` 与 B 的前辅音 `b` 的自然位置发生重叠。

- **期望（适度靠近）**：A 的**元音 `a` 先让缩短**，把 `s` 左移腾位；B 的 `b` 落在 `s` 之后**相接不重叠**。音素带上 `…a s b u…` 依次相接。
- **期望（很近，元音让尽）**：A 的元音 `a` 压到接近 0 后，**`s` 与 `b` 一起等比压短**（两者按各自标称长度等比缩，仍相接不重叠）。
- 对照（错误结果 / 旧 bug）：B 的前辅音 `b` **直接叠在 A 的 `s` 上**（两者占同一段时间、重叠），元音 `a` 不让 → 说明跨 note 簇压缩失效（per-note 封顶的老毛病）。
- **自由 note 同理**：把 A、B 都改成**未锁定**的自由歌词（`kas`/`bus`），合成。插件自带参考布局走同一算法，故自由 note 之间也应出现同样的「元音先让 → s+b 簇压」，**不再叠**。

## 用例 14 —（前置辅音簇 IsLead）多前置辅音整簇左展、显式归属

画歌词 note A = **`skat`**（前置簇 s,k + 元音 a + 后辅音 t），合成。

- **期望**：音素带 `s` `k` 在 A 头**前依次越界**（s 在 k 之前、各占 kLeadIn）、`a` 填充、`t` 占组末。
- **拖 A 头最左的 `s` 起点向左**：`s` 变长、整簇向 note 前自由伸展；`k`/`a`/`t` 不被挤占。
- **跨 note 簇压**：A=`skat` 紧邻 B=`pra`（前置簇 p,r）。靠近时 A 元音先让，再 `t`+`p`+`r` 簇一起等比压、不互叠。
- 判定：前置归属由 `IsLead` 显式标注（不再用 w>0 猜），多前置辅音作为一簇左展 / 整体参与压缩。

## 用例 15 —（核起点固定锚）后 note 压缩时核起点纹丝不动

画歌词 note A = **`ka`**，合成并**锁定**。在 A **紧邻其后**画 B（如 `bus`）并靠近 / 压缩。

- **期望**：A 的核 `a` 起点**恒在音符头**（拍点）、纹丝不动；前辅音 `k` 也不被推动。压缩只发生在核**尾部**——元音 `a` 从尾收缩让位、辅音簇压缩。
- **核起点不可拖**：试拖核起点（首个非前置音素的起边界）→ **无响应**（核恒在音符头，是固定拍点）。
- 对照（错误结果）：后 note 压过来时**把核也推走了**（核起点左移）→ 说明核起点没被当固定锚。

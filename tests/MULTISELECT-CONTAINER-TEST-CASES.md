# 多选下容器属性的三态合并 — 真机测试用例

多选音符时，note 属性面板里的容器控件（数组 `phonemes`/`pair`、变长键控对象）按统一规则合并：

- **数组**按 **index 对齐**：长度取所选音符里最长的那个；某 index 各音符全有且全等 → 显示该值（可编辑、扇出写所有音符）；
  值不等或有音符缺该位 → 该位显示 **Multiple**（与标量多选的 Multiple 同一呈现）。
- **对象 / 变长键控对象**按 **key 并集**：合并所有音符出现过的键；某键各音符全有且全等 → 显示该值；
  不等或缺于部分音符 → Multiple。
- **写**只落到「拥有该位 / 该键」的音符，绝不向更短的音符凭空补元素（不造数据错位）。

数据核心已由单测覆盖（`tests/TuneLab.Tests/MultiSelectMergeTests.cs`，14 例）；本文件测 GUI 交互真机回归。

**前置**：同 `PROPERTY-ARRAY-TEST-CASES.md`（装 `v1-suite.tlx`、声库 `[v1-suite] Conditional`）。
新建若干音符，在 note 面板 letters 框给不同音符设不同字母以制造不同的 phonemes seed/元素数。

## 数组多选（note 面板 phonemes / pair）

| 用例 | 操作 | 验证点 |
|---|---|---|
| 等长同值 | 选 2 个 phonemes 完全相同的音符 | 各行正常显示该值；编辑某行 → 两音符同步改 |
| 等长不同值 | 选 2 个同长但某位不同的音符 | 不同的那行显示 Multiple（dash/空），同值行正常；改该行 → 两音符都被覆盖成新值 |
| 不等长 | 选 phonemes 长 2 与长 3 的音符 | 显示 3 行（取最长）；第 3 行必为 Multiple（短音符缺该位）；前两行按等长规则三态 |
| 扇出写不补短 | 在「仅长音符有」的第 3 行编辑提交 | 只有原本有第 3 位的音符被改；短音符**不**被补出第 3 个元素 |
| 添加 | 点列表 `+` 选 Phoneme | 所有选中音符各自尾部追加一行（各加一个，不要求对齐绝对 index） |
| 删除 | 悬浮某行点 ✕ | 拥有该 index 的音符删该位；缺该位的音符不受影响 |
| 定长数组 pair | 选 pair 值不同的音符 | 2 行滑条；不同的位显示 Multiple，拖动覆盖所有选中音符 |
| undo/redo | 上述增删改后 Ctrl+Z/Y | 一次撤销单元整体回退/重做所有被扇出的音符 |

## 对象多选（note 面板 vibrato 等嵌套对象）

| 用例 | 操作 | 验证点 |
|---|---|---|
| 子字段三态 | 选 vibrato.depth 不同的音符 | depth 显示 Multiple，其余同值子字段正常；改 depth 扇出所有音符 |
| 深层嵌套 | 选 vibrato.lfo.range.min 不同的音符 | 逐层导航进去，min 显示 Multiple、扇出写正常 |

> 当前夹具把 Mixed Speakers 放在 part 面板。如把变长键控容器放进 note 面板多选测试：

| 用例 | 操作 | 验证点 |
|---|---|---|
| 键并集 | 选 speaker 集合不同的音符 | 显示所有音符出现过的 speaker 的并集 |
| 公共键编辑 | 编辑所有音符都有的某 speaker 的子值 | 扇出写所有音符 |

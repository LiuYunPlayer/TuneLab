# 合成参数回显（SynthesizedParameters）测试用例

> 本文只覆盖**本次改动的受影响范围**：引擎合成时产出的参数曲线（`ISynthesisSession.SynthesizedParameters`，
> 按轨 id 键、与音频/音高同一秒时间系、分段）在参数栏被**只读回显**——叠加在同名 voice 自动化轨上，
> 沿用合成音高回显的白色半透明视觉约定，区别于 config 色的可编辑曲线。
> 这是只读叠加层：不进数据层、不可编辑、不参与序列化。
> 轨集合显隐、孤儿数据、合成正确性等见各自基线/条件轨测试文档，不在此重复。
> 每个用例：**做什么 → 预期看到什么**。

## 准备

```bash
dotnet build TuneLab.sln -c Debug
dotnet build tests/TestPlugins.slnx -c Debug
powershell -File tests/pack-tlx.ps1          # 打成 tests/tlx/*.tlx
```

- 安装：把 `tests/tlx/v1-voice.tlx` 拖进 TuneLab 窗口（或扩展侧栏 → Install Extension）。
- 涉及的 UI 显示名：
  - **声库**：`Alice (V1 Test)` 或 `Bob (V1 Test)`（v1-voice）。
  - part 属性面板勾选框 **Enable Growl**（默认勾选）；勾选才暴露自动化轨 **Growl**（0..100，橙色 `#E5A573`）。
- 参照实现的回显数据：v1-voice 引擎每合成一块，就为该块产出一条确定性 **Growl** 回显曲线（一条 10..90
  的正弦波，每个合成块一段）。它**只在合成完成后出现**，触发合成即播放该 part / 走调度。

---

## 一、基本回显

### 1. 回显出现
- 选一条用 `Alice (V1 Test)` 的 MIDI part，写几个 note，保持 **Enable Growl** 勾选。
- 在底部参数栏点亮 **Growl** 轨。
- 触发合成（播放经过该 part）。每个合成块完成后 → **Growl 轨上出现一条白色半透明曲线**（10..90 的正弦起伏），
  即引擎产出的 Growl 回显；与橙色的可编辑 Growl 曲线（若已画）叠加显示、互不覆盖。

### 2. 分块/分段断开
- 写两组离得开的 note（中间留明显间隙，分成两个合成块）。合成后 →
  回显曲线**按块各自成段**，**段间断开**（间隙处不连线），不跨段拉直线。

### 3. 与可编辑曲线并存
- 在 **Growl** 轨画一条明显的橙色可编辑曲线（如前段 0、后段 80）。
- 合成 → 白色回显曲线与橙色可编辑曲线**同时可见**：白色是引擎产物（只读），橙色是你画的（可编辑）。
- 拖动橙色锚点 → 橙色随手变，**白色回显不动**（只读、不受编辑影响，直到下一次合成刷新）。

---

## 二、随轨显隐 / 视图联动

### 4. 回显随轨显隐
- 合成出回显后，取消勾选 **Enable Growl** → Growl 轨从参数栏消失，**回显白线一并消失**（宿主只绘可见轨）。
- 重新勾选 → Growl 轨回来，**回显白线重新出现**（数据仍在引擎侧，无需重新合成）。

### 5. 切换激活轨
- 参数栏再加一条别的可见轨（如内置 Volume），在 Growl 与该轨间切换激活。
- 无论 Growl 是否为当前激活轨，只要它**可见**，其回显白线都应绘制（激活态线宽更粗的是可编辑曲线，回显恒为细白线）。

### 6. 缩放/横向滚动跟随
- 横向缩放、左右滚动钢琴窗 → 回显白线**随时间轴正确缩放/平移**，与音频段、可编辑曲线对齐，不漂移。

---

## 三、不回归

### 7. 无回显轨不受影响
- 内置轨（Volume 等）与其他没有回显数据的轨：参数栏绘制**与改动前一致**，无多余白线、无异常。

### 8. 静态/空回显插件不回归
- 用 `v1-suite` 或 `v1-i18n` 声库（其 `SynthesizedParameters` 为空）：参数栏**无任何回显白线**，绘制与改动前一致。

---

## 备注（验证侧重点）

- 回显是**只读叠加层**：不写数据层、不可编辑、不进工程序列化（重开工程后未重合成则无回显，属预期）。
- 时间系是**秒**（与音频/音高产物一致），宿主按 tempo 换算到 tick 再到屏幕 x；用例 6 验证换算正确。
- 视觉约定沿用合成音高回显（白色半透明），与 config 色的可编辑曲线区分；颜色/透明度为宿主侧渲染选择，可后续微调。

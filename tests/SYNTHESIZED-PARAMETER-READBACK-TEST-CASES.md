# 合成参数回显（只读回显轨）测试用例

> 本文覆盖**本次改动的受影响范围**：引擎合成时产出的参数曲线被宿主作**一等只读回显轨**呈现。
> 轨形态由引擎经 `ISynthesisSession.GetSynthesizedParameterConfigs(context)` 声明（独立 key、自带 DisplayText/Min/Max/Color、
> 分段形 `DefaultValue=NaN`），曲线数据经 `SynthesizedParameters` 按同一批 key 承载。
> 宿主把这些轨：
> - 显隐开关放在**参数区标题栏右侧**（曲线区正上方那条细栏），复用参数栏的小眼睛图标，不进底部分源 tabbar；
> - 画成**曲线与参数区底部围成的半透明积分面积**（用各自 config 色、仅填充无描边），区别于可编辑轨的细线；
> - **只读**：不可编辑、不可激活、不进数据层、不参与序列化。
> 这是只读呈现层，**不再叠加到同名编辑轨**（旧版"按 id join 画白线"已废弃）。
> 轨集合显隐、孤儿数据、合成正确性等见各自基线/条件轨测试文档，不在此重复。
> 每个用例：**做什么 → 预期看到什么**。

## 准备

```bash
dotnet build TuneLab.sln -c Debug
dotnet build tests/TestPlugins.slnx -c Debug
powershell -File tests/pack-tlx.ps1          # 打成 tests/tlx/*.tlx
```

- 安装：把 `tests/tlx/v1-voice.tlx` 拖进 TuneLab 窗口（或扩展侧栏 → Install Extension）。
- 涉及的 UI 显示名：
  - **声库**：`Alice (V1 Test)` 或 `Bob (V1 Test)`（v1-voice）。
  - **回显轨**：`Energy`（粉色 `#E573B0`，0..100），由引擎**恒声明**——合成前 key 即存在。
    显隐按钮在**参数区标题栏右侧**（小眼睛 + "Energy" 文本）。
  - 可编辑轨（作对照）：连续轨 **Growl**（橙、勾 **Enable Growl** 才暴露）、分段轨 **Bend**（蓝）。
- 参照实现的回显数据：v1-voice 引擎每合成一块，就为该块产出一条确定性 **Energy** 回显曲线（一条 10..90 的
  正弦波，每个合成块一段）。它**只在合成完成后出现**，触发合成即播放该 part / 走调度。

---

## 一、声明与显隐（标题栏小眼睛）

### 1. 按钮合成前即存在
- 选一条用 `Alice (V1 Test)` 的 MIDI part，写几个 note，**先不合成**。
- 看参数区标题栏右侧 → 已有一个 **Energy** 按钮（小眼睛 + 文本）。回显轨是预声明的，不依赖合成才出现。
- 默认**未点亮**（眼睛灰白、文本偏暗），参数区此时不画回显。

### 2. 点亮 / 熄灭
- 单击 **Energy** 按钮 → 点亮（眼睛变粉色 config 色、文本变亮）。再次单击 → 熄灭。
- 点亮后触发合成（播放经过该 part）→ 每个合成块完成后，参数区出现**粉色半透明积分面积**（曲线到底部的填充区，仅填充无描边），
  即 Energy 回显。熄灭 → 填充区立即消失（数据仍在引擎侧，无需重新合成；重新点亮即恢复）。

### 3. 换声源按钮立即刷新
- 把该 part 的声源从 `Alice (V1 Test)` 换成 `Bob (V1 Test)`（或换成空声源再换回）→ 标题栏 **Energy** 按钮**立即重绘**
  （不必把鼠标移上标题栏才出现/消失）。

### 4. 标题栏手势共存
- 在标题栏**空白处**（非按钮）按住上下拖动 → 参数区高度随之改变（沿用拖拽改高）。
- 在 **Energy** 按钮上按下 → 只切换显隐，**不触发拖拽改高**（高度不变）。

---

## 二、绘制

### 5. 积分面积 + 分段断开
- 点亮 Energy，写两组离得开的 note（中间留明显间隙，分成两个合成块）。合成后 →
  每个合成块各自一块**独立的填充面积**，**段间断开**（间隙处既不填充也不连线），不跨段拉平。

### 6. 与可编辑轨并存比对
- 点亮 Energy 回显轨；同时在底部 tabbar 点亮可编辑轨（如 **Growl**，并画一条明显的橙色曲线）。
- 合成 → 粉色 Energy 填充面积与橙色 Growl 细线**同屏叠加**：面积是引擎产物（只读），细线是你画的（可编辑）。
- 拖动 Growl 锚点 → 橙线随手变，**Energy 填充面积不动**（只读、不受编辑影响，直到下一次合成刷新）。

### 7. 只读、不可激活
- 回显轨**不在底部 tabbar**、不能设为激活编辑轨；在参数区对着填充面积做绘制/擦除/拖锚点等编辑手势 →
  **无任何效果**（操作落到当前激活的可编辑轨或空操作），Energy 面积不被编辑。

### 8. 缩放 / 横向滚动跟随
- 横向缩放、左右滚动钢琴窗 → Energy 填充面积**随时间轴正确缩放/平移**，与音频段、可编辑曲线对齐，不漂移。

---

## 三、不回归

### 9. 无回显轨的声库
- 用 `v1-suite` 或 `v1-i18n` 声库（其 `GetSynthesizedParameterConfigs` 为空）：
  标题栏**无回显按钮**，参数区**无任何积分面积**，绘制与改动前一致；标题栏空白处仍可拖拽改高。

### 10. 可编辑轨不受影响
- 内置轨（Volume 等）与可编辑 Growl/Bend：参数栏绘制**与改动前一致**（细线、可编辑、可激活），
  不因回显改动出现多余填充或异常。

---

## 备注（验证侧重点）

- 回显是**只读呈现层**：不写数据层、不可编辑、不进工程序列化（重开工程后未重合成则无回显数据，属预期；
  但 chip 因预声明仍在）。
- 时间系是**秒**（与音频/音高产物一致），宿主按 tempo 换算到 tick 再到屏幕 x；用例 7 验证换算正确。
- 视觉：积分面积 = 曲线与参数区底部围成的填充区，用轨 config 色、低透明度（约 0.25）、**仅填充无描边**；
  颜色/透明度为宿主侧渲染选择，可后续微调。

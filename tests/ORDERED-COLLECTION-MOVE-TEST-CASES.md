# 有序集合维序 / 统一 Move 测试用例

覆盖范围：本次数据层重构——note / vibrato / part 三类**就地改排序键（pos/dur）**的有序集合统一走 `Move`（摘除→改→按新键重插维序），vibrato 由 `DataObjectList` 改为有序链表，以及删除归属统一（链表反向指针 O(1) 校验 + DEBUG 断言）。**仅测本次受影响范围**；音符重叠创建/钳位、agent part 工具、分段轨等基线各有其文档，不在此重测。

不涉及任何插件，**无需打 tlx / 备插件**。空工程手画内容即可测。

## 测试前置

1. **构建 Debug**：`dotnet build TuneLab/TuneLab.csproj -c Debug`（启动前先确认没有别的 TuneLab 实例占单实例锁）。
   - **必须 Debug**：归属校验是 `Debug.Assert`，只在 Debug 生效。测试期若弹出断言对话框（"RemoveNote/RemoveVibrato/RemovePart: …不属于…"），即抓到一个真实归属 bug——记录复现步骤后报告。
2. 启动应用，新建空工程。
3. 准备内容（手动画即可）：
   - 一条 midi part，内画 **若干音符**，其中**至少两个同起点、不同长度的"和弦"音符**（验证同 pos 时按时长的次序）。
   - 用颤音工具画 **2~3 个颤音**（位置不同，至少两个相邻）。
   - 轨道窗里一条轨上放 **2~3 个起点不同的 part**，另备一条空轨用于跨轨拖动。
4. 每个操作后用 **Ctrl+Z 撤销 / Ctrl+Y 重做**，确认整步一次性回退、再一次性重做，结果与操作前/后一致。

> 维序是否正确多为内部状态，观察代理：①拖/缩/移后视觉位置正确、无音符/颤音/part 消失或重影；②**改完后在被动过的元素附近再新画一个**，新元素应落在正确次序处（次序乱了新插入会错位）；③保存→重开后位置不变；④全程无断言对话框、无崩溃。

## Bug 2：note 维序

| # | 操作 | 预期 |
|---|---|---|
| N1 | 拖动单个音符到新的 tick 位置（含越过相邻音符） | 音符落到新位；越过的邻居次序正确；其后在它两侧各补画一个新音符，新音符落位正确。撤销/重做一步到位 |
| N2 | 框选多个音符整体平移（含改变它们与未选音符的相对次序） | 整组平移、彼此与外部次序均正确；撤销一步回退全部 |
| N3 | 拖音符**左边界**（改 pos+dur，起点移动） | 起点移动、末端不动；越过左邻时次序正确；撤销回退 |
| N4 | 拖音符**右边界**（只改 dur，起点不变） | 末端伸缩、起点不动；视觉与次序正确；撤销回退 |
| N5 | **和弦同起点**：把两个同起点音符之一拉长/缩短，使两者长度大小关系反转 | 两音符仍正确渲染、无重影；同起点按时长的次序自洽（长的在前）；其后在该起点再画一个音符，落位正确；撤销回退 |
| N6 | 波形/相邻边界拖动（两相邻音符之间的分界，左缩尾、右移头） | 左音符缩尾、右音符起点随分界移动；两者次序正确；把分界拖到与某音符完全重合时该音符被删除；撤销回退 |
| N7 | 对某音符右键"在此分割"（SplitAt） | 原音符缩到分割点、右半成为新音符；两者次序正确、紧邻；撤销一步同时退回新音符与原长度 |
| N8 | "去除重叠"命令（RemoveOverlaps）作用于一组重叠音符 | 重叠被消除（缩尾/删除）；保留音符次序正确；撤销一步回退 |

## Bug 2：part 维序

| # | 操作 | 预期 |
|---|---|---|
| P1 | 同轨内拖动 part 改起点（含越过相邻 part） | 落到新位、轨内 part 次序正确；其后在附近建新 part 落位正确；撤销/重做一步 |
| P2 | 拖 part **右边界**改时长（PartEndResize，走 MovePart） | 末端伸缩、起点不动；撤销回退 |
| P3 | **跨轨**拖动 part 到另一条轨（换父路径，保留显式 remove/insert） | part 移到目标轨、源轨移除、目标轨内按起点正确排序；撤销一步回到原轨原位 |
| P4 | 多选 part 跨轨整体拖动 | 全部迁移、各目标轨内排序正确；撤销一步全回退 |

## Bug 2：脚本 / Agent 改 pos/dur（reorder 标志已删，统一走 Move）

| # | 操作 | 预期 |
|---|---|---|
| S1 | 脚本 `tl.setNote(note, {pos: …})` 改音符起点致次序变 | 音符移位且列表有序；脚本结束后一次撤销整体回退 |
| S2 | 脚本 `tl.setNote(note, {pitch: …})` **只改音高**（非排序键） | 音高变、位置不变、无多余重排副作用；撤销回退 |
| S3 | 脚本 `tl.setPart(part, {pos: …})` / `{dur: …}` | part 移位/伸缩且轨内有序；回灌/后续操作正常 |
| S4 | Agent `set_note` / `set_part_properties` 改 pos/dur | 同上；编号随排序变化正确（part），撤销为一步 |
| S5 | 脚本对同一已删 handle 再次 `removeNote` | 报"handle was removed"错误（Removed 守卫拦截），**不应**触发归属断言或崩溃 |

## Bug 1：删除归属（DEBUG 断言不应在正常操作中触发）

| # | 操作 | 预期 |
|---|---|---|
| D1 | 选中并删除音符 / 颤音 / part（Delete 键、菜单、删除选中） | 正常删除，撤销恢复；**无断言对话框** |
| D2 | 全过程中（N/V/P/S 各用例）观察 | 任何拖动/缩放/分割/移动中途都不应弹出 `RemoveNote/RemoveVibrato/RemovePart` 断言；若弹出即记录步骤报告 |

## 回归（确认未破坏既有路径）

- R-1：保存工程→重开，音符/颤音/part 的位置与次序全部一致。
- R-2：撤销/重做穿过若干次 Move 操作，工程状态始终自洽（无错位、无重影、无崩溃）。
- R-3：参数同步模式（Settings.ParameterSyncMode）开启时拖动音符，音高/自动化曲线随音符一起平移（与改前一致）。
- R-4：纯非键编辑（改音高/歌词/part 名）不引起元素在列表中位置跳动，撤销为一步。

# 合成音素显示 / 失效 / 门控 — 手测用例

只覆盖本轮改动：**合成音素的显示生命周期**——粗/细线区分、移动即失效清除（引擎驱动）、相接待合成邻居的显示门控、显示侧防御去重叠、只锁被拖 note。
钉死音素的时长 / 布局 / 跨 note 压缩 / 旧工程兼容基线见 `PHONEME-ANCHOR-TEST-CASES.md`；voice 单声部去重叠见 `VOICE-NOTE-OVERLAP-TEST-CASES.md`。不重复其内容。

## 模型要点（判定期望时对照）

- **清除责任在引擎**：note 的位置 / 长度变化 → 引擎把受影响块标脏 → 脏 / 合成中的块**不报告**其合成音素 → 宿主据此把对应 note 留白，待重合成回显。宿主不在数据层单点清除（故相接的邻居若同属一脏块，也一起留白）。
- **显示侧防御去重叠**：宿主显示时把固定 + 合成音素都过同一个跨 note 去重叠（`DisplayPhonemes`）。即便插件未及时清除 / 未自行去重叠，宿主显示也不重叠（无重叠时为 no-op，故守约插件显示不变）。
- **显示门控**：本 note 的最终音素布局依赖相接邻居的音素几何。某侧有「相接、非乘客（非延音符 `-`）、却尚无音素」的邻居（如正在合成 → 留白）时，本 note **一并留白**、待邻居就绪。有空隙 / 到 part 末尾则各自独立、照常显示。（「相接、有歌词、却永久不返回音素」属插件违约，由插件自负，宿主不为它兜底。）
- **粗 / 细线**：固定音素刻度线**粗、实色**；合成音素刻度线**细、半透明**——一眼区分某 note 是固定还是合成。
- **只锁被拖 note**：在音素带拖某 note 的音素，只把**该 note** 锁为钉死，邻居保持合成态（仍会随编辑重新预测），不冻结全曲。

> 内部改动（不单列用例，顺带确认）：合成产物更新信号已按 **音素 / 参数 / 音高 / 状态** 分离。验证时只需留意合成过程中状态进度条、音素带、参数回显轨都照常刷新即可。

## 前置准备

1. 关掉 TuneLab 再开（取最新宿主）；v1-voice 已自动装好，声库 UI 名 **「Alice (V1 Test)」**。
2. 新建空工程 + 一条 MidiPart 挂「Alice (V1 Test)」。歌词解析：元音 = a/e/i/o/u；如 `la`=l+a、`ka`=k+a、`skat`=s,k/a/t。`-` 是唯一延音符（乘客）。
3. **关键**：测试插件内置约 **1 秒合成延时**，用于肉眼区分「合成中（留白）→ 合成完（回显）」两态。看不到留白说明延时被关（插件常量 `kSimulatedRenderMs`）。
4. 音素带在钢琴窗下方波形区，需横向放大到一定比例才显示。

## 用例 1 — 粗 / 细线区分固定 vs 合成
画一个歌词 note，合成完成 → 音素刻度线**细、半透明**（合成）。在它上面拖动一下任一音素 → 该 note 刻度线变**粗、实色**（已锁为固定）。
- 判定：粗细 / 明暗能区分固定与合成。

## 用例 2 — 合成延时两态可见
画一个歌词 note `la`，观察提交后：约 1 秒内音素带**留白**、状态带显示**合成中 / 进度**；约 1 秒后音素**回显**。
- 判定：延时生效，能看到合成前后两态（后续用例都靠这个观察留白）。

## 用例 3 — 移动 note 即清除合成音素
合成好一个孤立 note A。把 A **整体平移**到别处空旷位置。
- 期望：A 的合成音素**立即留白**（不残留旧音素），约 1 秒后在新位置**重新回显**。
- 对照（错误）：移动后仍显示旧位置音素 / 与新位置错位 → 引擎标脏清除失效。

## 用例 4 — 相接双方一起清除
画两个**相接**的合成 note A、B（如 `ka`、`la`），合成完。把其中一个稍挪（仍与另一个相接 / 重叠）。
- 期望：**A、B 双方**音素一起留白（同一脏块），约 1 秒后一起回显。
- 判定：移动影响的不只被移动的 note，相接邻居（同块）一并失效——不会出现一方留白、另一方还挂着旧音素。

## 用例 5 — 显示门控：相接的待合成邻居 → 本 note 也留白
拖一下某歌词 note A 的音素，使 A 成为**固定音素**（粗线）。在 A **紧邻其后**画一个有真歌词的 note B（如 `la`）。
- 期望：B 合成那约 1 秒内 B 留白，**A 也跟着留白**（A 右边界依赖 B 的音素几何，未决）；B 合成完后**两者一起显示**。
- **有空隙不门控**（对照）：把 B 拖到与 A **留一段空隙**。B 合成期间 A **照常显示**（无相接依赖）。
- 对照（错误）：A 在 B 完成前就按「无邻居」显示、B 完成后又跳变 → 门控失效。

## 用例 6 — 显示侧去重叠（首条 bug 的回归）
固定 note A（如 `la`），在其后**相接**画合成 note B（带前辅音，如 `ka`），等 B 合成完。
- 期望：B 的前辅音 `k` 越界进 A 元音尾的位置，显示上 **A 元音让位、不与 `k` 重叠**（固定 + 合成统一过跨 note 去重叠）。
- 对照（错误·首条报告）：A 元音与 B 的 `k` 在显示上直接叠在一起。

## 用例 7 — 只锁被拖 note，邻居保持合成
画三个相接合成 note A、B、C，合成完。在 **B** 上拖一下音素锁定。
- 期望：**只有 B** 变粗线（固定）；A、C 仍细线（合成）。此后移动 A 或 C 仍会重新预测（合成态未被冻结）。
- 已知代价：提交 B 的编辑触发重合成时，与 B 同块的相接邻居在那约 1 秒内会留白再回显（可接受，换不冻结全曲预测）。
- 对照（错误·已废弃）：拖 B 把整条 part 的 note 全锁成固定（A、C 也变粗）→ 又在全锁。

# Voice Note 去重叠（宿主单声部 + 暗色显示）— 手测用例

只覆盖本需求受影响范围：**voice part 的 note 重叠由宿主非破坏地去重叠**——

- **有效结束**：宿主派生 `EffectiveEnd = min(本 note 末, 下一 note 起点)`（后盖前），喂插件的 note 天然不重叠、单声部。
- **非破坏**：note 的 `Dur` 不改（切到 instrument 引擎可恢复多声部）；被砍掉的尾段在钢琴窗**暗色盖住**显示，亮色 = 真正发声段。
- **当前无条件**：instrument 引擎尚未存在，故所有 voice part 都去重叠；将来挂 instrument（保留重叠多声部）再按引擎类型 gate。

> 变更说明：旧版「V1 原样多声部 / Legacy 单声部」的差异**已取消**——去重叠责任从插件搬到宿主，**两个声库现都收到去重叠后的单声部 note**，听感应一致。
>
> 注意：钢琴窗自由重叠（拉伸/移动不再截断邻居），但**只有 `.tlp` JSON 工程能注入重叠 note**（MIDI 导入去重叠、JSON 原样读入）。故用夹具 `fixtures/voice-note-overlap.tlp`。

## 前置准备

1. **拿新宿主二进制**：先**关掉运行中的 TuneLab**，再跑 App（自动 build 最新宿主，含去重叠 + 暗色显示）。
2. **安装两个测试插件**（拖进窗口或扩展侧边栏 Install Extension）：
   - `tests/tlx/legacy-voice.tlx` → 老接口声库，UI 显示名 **「Carol (Legacy Test)」**
   - `tests/tlx/v1-voice.tlx` → V1 会话声库，UI 显示名 **「Alice (V1 Test)」**
3. **打开夹具**：导入/打开 `tests/fixtures/voice-note-overlap.tlp`（两条轨分别挂上面两个声库，note 完全相同便于 A/B）。

工程速览（120 bpm，480 tick = 0.5 s）：

| note | pos (tick) | dur (tick) | pitch | 用途 |
|---|---|---|---|---|
| 1 | 0 | 720 | C4(60) | 尾部重叠：被 note2 盖 |
| 2 | 480 | 720 | E4(64) | 起点落在 note1 内 |
| 3 | 1920 | 720 | C4(60) | 同起点和弦：最长 |
| 4 | 1920 | 480 | E4(64) | 同起点和弦：中 |
| 5 | 1920 | 240 | G4(67) | 同起点和弦：最短 |
| 6 | 2880 | 480 | D4(62) | 和弦后的独立 note |

## 用例 A — 尾部重叠（后盖前）

note1 C4 [0–0.75s) 与 note2 E4 [0.5–1.25s) 在 [0.5,0.75s) 重叠。

- **听感（两轨一致）**：note1 的尾被钳到 note2 起点 → C4 仅 [0–0.5s) 发声，随后 E4 [0.5–1.25s)。**单声部、重叠区无叠音**。
- **显示**：note1 矩形 `[0.5s, 0.75s)` 那段**暗色盖住**（被 note2 重叠覆盖），`[0, 0.5s)` 亮色（真正发声）。

## 用例 B — 同起点和弦塌成单音

note3/4/5 同起点 1920（2.0s），数据层序按 EndPos 降 = C4→E4→G4。

- **听感（两轨一致）**：C4、E4 的有效结束被钳到同起点（归零、不发声），只剩排最后、最短的 **G4 [2.0–2.25s)** 存活；随后 D4 [3.0–3.5s)。和弦塌成单音 G4。
- **显示**：C4 矩形 [2.0–3.0s) 与 E4 矩形 [2.0–2.5s) **整段暗色**（有效时长归零）；G4 [2.0–2.25s) 亮色；D4 正常亮色。

## 判定

| | 听感 | 显示 |
|---|---|---|
| 用例 A 重叠区 [0.5,0.75s) | **无叠音**，先 C4 后 E4 | note1 尾段暗色、前段亮色 |
| 用例 B 同起点和弦 | 塌成单音 **G4** | C4/E4 整段暗色，仅 G4 亮色 |

两轨（Carol / Alice）**听感应一致**——差异不再来自插件多/单声部，而是宿主统一去重叠。暗色段直观标出"被重叠盖掉、不发声"的部分。

> 单测侧：`legacy/compat/.../NoteOverlapClampTests.cs` 仍钉死 legacy compat 视图层的后盖前钳位算术（喂老引擎的冗余兜底，现宿主已先去重叠故退化为恒等，保留无害）。

# 音素编辑器测试（独立用例）

侧栏 Phoneme 面板从"只读 + 引擎属性编辑"扩为"自由音素编辑器"。分阶段落地，本文档逐步追加。

## 前置

- 已 build 主工程并重开 TuneLab（本阶段纯宿主 UI 改动，**插件未变、无需重打包**）。
- 测试夹具 `V1 Test Voice`（已装）：每个音素声明 `accent`（滑条）+ `Offset (ms)`（可拖数值框）两条属性。

---

## 第 1 步：门控改 + 符号可编（当前）

进入：新建工程 → part 选 **V1 Test Voice** 任一声库 → 画带歌词音符（如 `ka`、`skat`）→ 等合成 → 选中含音素的音符 → 右侧 **Phoneme** 页签。

| # | 操作 | 期望 |
|---|---|---|
| 1 | 选中一个有音素的音符 | 每个音素一行；最左是**符号块**（不再是只读标签），其右是 accent/Offset 控制器。 |
| 2 | **双击**某音素的符号块 | 原地出现输入框、全选当前符号。 |
| 3 | 键入新符号回车（或点别处） | 符号更新；该音符若原为合成音素则被固化（LockPhonemes），触发重合成；音素带同步显示新符号。 |
| 4 | 双击后不改、直接回车 | 不提交、不产生多余撤销步。 |
| 5 | 改符号后 **Ctrl+Z** | 一步回退到原符号。 |
| 6 | 多选多个含同位音素的音符，改某 slot 符号 | 扇出到所选各音符该位；原符号不一致的 slot 显示 `(...)`，改后统一。 |
| 7 | 选中**仅有合成音素、引擎也没声明属性**的情形（理论上）| 面板仍出行、符号可编（门控不再要求引擎声明属性）。注：V1 夹具每音素都声明了属性，此项需无属性引擎验证，可暂略。 |

---

## 第 2 步：顶部色条 + 时长/权重值框（当前）

每个音素 = **顶部色条**（IsLead 配色：符号靠左 + 时长/权重值框靠右，无文字标签、悬浮 tooltip）+ 其下（引擎声明属性时才有）自定义属性面板。

| # | 操作 | 期望 |
|---|---|---|
| 1 | 看某音素 | 顶部色条左侧为符号、右侧两个数值框：时长（显示如 `120 ms`）+ 权重（如 `1.00`）；悬浮分别提示 `Duration` / `Stretch Weight`。引擎属性（accent/Offset）在色条下方；无引擎属性时色条下方为空、不留空白框。 |
| 2 | 在时长框横向拖动 | 数值实时变化（1ms/px）、右增左减、下界 0 无上界；**钢琴窗音素带拖动中实时跟手**（辅音权重 0 时长直接变化）。 |
| 2b | 拖权重框 | 钢琴窗音素带布局实时跟手（元音让位/辅音压缩随权重变）。 |
| 3 | 双击时长框键入数字（ms）回车 | 提交为该值；合成音素首次编辑即固化（Lock）。 |
| 4 | 拖/改 时长 后 Ctrl+Z | 一步回退。 |
| 5 | 改权重框（拖/键入） | 扇出 + 固化 + 一个撤销步；默认 1、下界 0、2 位小数。 |
| 6 | 多选含同位音素的音符改时长/权重 | 扇出到各音符该位；原值不一致时框显 `-`，改后统一。 |
| 7 | 长符号（如自定义多字符音素） | 符号靠左占整行宽度容纳，不挤压。 |
| 8 | 存盘重开 | 时长/权重持久（固化为钉死音素后随工程存储）。 |

注：元音(权重>0)的时长被布局忽略（恒按填充派生），改它有记录但音素带无可见变化——属预期。

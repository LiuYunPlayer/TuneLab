# 音素双列表 + BodyOffset 手测用例

> 只测本次升级（`Preutterance` 标量 → `LeadingPhonemes`/`BodyPhonemes` 双列表 + 有符号 `BodyOffset`）受影响的范围：
> 分类抗抖、跨拍归属、`BodyOffset=0` 精确、lead-in 空白、侧栏合并对齐不错位、序列化往返、legacy 兼容。
> 不重跑已通过的音素编辑方向 / roll 基线（见 `PHONEME-EDIT-DIRECTION-TEST-CASES.md`，其行为经本次内部等价参数化后不变）。

## 前置

- build + pack 样例声源插件 `V1.Voice`（UI 显示名见插件 manifest，非 Type id）；开应用、新建 voice part、选该声源。
- 关键测试入口：歌词加前缀 `*` = 跨拍测试（令 note 头切进首音素）；波形带下半区 = 音素带（明=主体、暗=引导）；侧栏 Phoneme 面板 = 逐音素符号/时长/权重 + 多选合并对齐。

## 1. 分类抗抖（核心动机）

1.1 输入一个 `ka`（引导 k + 主体 a）。波形带：k 段显**暗**（引导）、a 段显**亮**（主体）。
1.2 拖 k|a 分界线（波形带音素起边界柄），令分界在音符头附近来回微动 ±1 帧。
   - **期望**：k 恒暗、a 恒亮，**分类绝不翻转**（旧模型此处会因帧量化翻 lead/body、连带明暗与侧栏对齐一起翻）。
1.3 侧栏多选 `ka` + `ta`（都是引导辅音 + 元音 a）：k、t 对齐到同一「引导 slot（-1）」，a、a 对齐到「核 slot（0）」。微调时长使分界抖动，对齐 slot 不错位。

## 2. 跨拍归属（几何分不了、必须靠列表）

2.1 `*ka`：k 跨音符头（前半在拍前、后半在拍后），但 k 仍属**引导**（暗）。波形带 k 段横跨音符头竖线。
2.2 `*ai`（无引导辅音、元音 a 起手）：a 跨音符头，a 属**主体**（亮）。
   - **对比 2.1**：2.1 跨拍音素归引导、2.2 跨拍音素归主体——`(start,end,音符头)` 几何相同、分类相反，只有双列表能区分。

## 3. BodyOffset=0 精确落头

3.1 普通 `ka`（无 `*`）：主体 a 的起点**精确** == 音符头（无亚帧偏移）。放大到帧级看 a 起点竖线与音符头竖线严丝合缝。
3.2 纯元音 `a`（无引导）：a 从音符头起、填到音符末，无引导段。

## 4. lead-in 空白（不钳，Q2）

4.1 构造 `BodyOffset` 大于引导总长的情形（拖首引导音素分界线尽量右移越过音符头）：允许音符头与首音素间出现一段空白/静音起手，**不报错、不强制钳位**。观感为一段起手静音即可（罕见边角）。

## 5. 侧栏合并对齐不错位

5.1 多选 `ska`（引导 s,k + 主体 a）+ `ta`（引导 t + 主体 a）：
   - 核 slot（a、a）对齐；`ska` 的 k 在 -1、s 在 -2；`ta` 的 t 在 -1。t 与 k 同 slot、s 独占 -2。
5.2 在某 slot 改时长 / 权重 / 符号、右键拆分 / 删除：只作用于**该 slot 各 note 对应列表内**的音素，归属（引导/主体）不变；删空回落合成音素口径。

## 6. 编辑生长方向（双列表使其自动化）

6.1 侧栏改**引导**音素时长 → 向左长（主体整体不动、`BodyOffset` 不变）。
6.2 侧栏改**主体**音素时长 → 向右长（`BodyOffset` 不变）。
   - 二者均无需旧「按拍前占比耦合 Preutterance」，方向随所属列表自动成立。

## 7. 序列化往返

7.1 钉死若干音素（含引导 + 主体 + 非零 `BodyOffset`），保存为 `.tlp`（JSON）与 `.tlpx`（CBOR），重开：两列表成员、各音素时长/权重/属性、`BodyOffset` 全保真。
7.2 打开一个 **legacy 1.x** 工程（旧 `startTime/endTime` 音素）：按区间中点 < 音符头分引导 / 主体、`BodyOffset=0`，逐字直显不吸头、观感与旧版一致。

## 8. legacy 声源兼容

8.1 用 legacy voice 插件（compat 层）合成：回显音素按引擎绝对位置分引导 / 主体、`BodyOffset = 主体首起点 − 音符头`（不吸头）；全 `w=0`、随音符等比缩放；无 Sil 接缝。

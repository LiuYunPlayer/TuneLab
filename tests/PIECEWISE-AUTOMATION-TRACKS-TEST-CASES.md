# 可编辑分段自动化轨（声明分段轨）测试用例

> 本文只覆盖**本次改动的受影响范围**：除内置 Pitch 外，声源/效果器可声明**可编辑分段轨**
> （`GetPiecewiseAutomationConfigs`，分段 + 段间空、无默认基线），宿主把它从声明→数据→渲染→编辑→存盘全链路打通。
> 编辑交互**参考 pitch**（绘制/擦除/锚点选/移/删/双击插点），值轴按轨 config 的 Min/Max 映射。
> 连续轨（含默认基线、vibrato 等）见其基线/条件轨测试文档，不在此重复；只读的合成参数回显见
> SYNTHESIZED-PARAMETER-READBACK 测试文档。
> 每个用例：**做什么 → 预期看到什么**。

## 准备

```bash
dotnet build TuneLab.sln -c Debug
dotnet build tests/TestPlugins.slnx -c Debug
powershell -File tests/pack-tlx.ps1          # 打成 tests/tlx/*.tlx
```

- 安装：把 `tests/tlx/v1-voice.tlx` + `tests/tlx/v1-effect.tlx` 拖进 TuneLab 窗口（或扩展侧栏 → Install Extension）。
- 本次涉及的分段轨（UI 显示名）：
  - **声库** `Alice (V1 Test)` / `Bob (V1 Test)`（v1-voice）：参数栏多一条分段轨 **Bend**（蓝 `#73C2E5`，范围 -100..100）。
  - **效果器** `TLTestGain`（v1-effect）：参数栏多一条分段轨 **Formant**（青 `#00C2A8`，范围 -100..100）。
- 分段轨与连续轨同处**钢琴窗底部参数栏**：同一来源（voice / 某 effect）的连续轨与分段轨在同一分组里
  （连续在前、分段在后），组间有竖分隔符。
- **重要范围说明**：本次只做 host 侧"可编辑 + 存盘 + 显示"。**引擎/DSP 暂不消费**这些分段轨
  （effect 分段轨回写、voice 分段轨参与合成均为后续需求）——所以编辑分段轨**不会改变音频**，属预期。

---

## 一、声明与显隐

### 1. 分段轨出现在参数栏
- 选一条用 `Alice (V1 Test)` 的 MIDI part → 参数栏能看到 **Bend** 轨按钮（在 voice 分组内、连续轨之后）。
- 给 part 加效果器 **TLTestGain** → 该 effect 分组里能看到 **Formant** 轨按钮。

### 2. 点亮 / 激活
- 点 **Bend** 按钮使其可见（Visible）→ 参数区出现该轨（初始为空、无曲线）。
- 再点使其激活（Edit 态）→ 成为当前编辑轨，参数区左侧显示其 Min/Max 标注（-100 / +100）。

---

## 二、编辑（参考 pitch；切到 Anchor 工具或对应绘制工具）

### 3. 绘制（默认绘制工具，左键拖）
- 激活 **Bend**，在参数区按住左键拖一段曲线 → 画出一段分段曲线（蓝色）；松手即提交。
- 在另一处再拖一段（与第一段不相连）→ 形成**两段**，**段间断开**（空隙处不连线）。

### 4. 擦除（右键拖）
- 在已画曲线上右键横向拖 → 拖过的区间被清空（该段被截断 / 删除），段间空。

### 5. 锚点工具：双击插点 / 拖动 / 框选 / 右键删
- 切 Anchor 工具，激活 **Bend**：
  - 在曲线段上**双击** → 插入一个锚点并可随手拖动。
  - **拖动**某锚点 → 该点随鼠标移动（值被钳在 -100..100），曲线实时更新。
  - 左键空白处**拖框** → 框内锚点被选中（高亮）；Ctrl 拖框累加选择。
  - **右键**某锚点 / 右键横拖 → 删除锚点 / 删除拖过区间的锚点。

### 6. 键盘
- 框选若干锚点后按 **Delete** → 选中锚点被删除。
- **Ctrl+A**（参数区聚焦时）→ 选中该分段轨全部锚点。

### 7. undo / redo
- 上述任一编辑都可 **Ctrl+Z / Ctrl+Y** 回退 / 重做，曲线状态正确还原。

### 8. effect 分段轨同样可编辑
- 激活 **TLTestGain** 的 **Formant** 轨，重复用例 3~7 → 行为一致（effect 分段轨走同一编辑路径，按 effect 索引归属）。

---

## 三、与连续轨并存、不串扰

### 9. 同组内连续 / 分段互不干扰
- voice 分组里既有连续轨（如 **Growl**，需勾 Enable Growl）又有分段轨 **Bend**：
  在两者间切换激活、各自编辑 → 互不影响；连续轨仍画含基线/vibrato 的最终曲线，分段轨画段间断开的曲线。

### 10. 切换激活轨保留各自数据
- 在 **Bend** 上画好曲线 → 切去激活别的轨 → 再切回 **Bend**：曲线仍在、原样。

---

## 四、存盘 round-trip

### 11. 保存 / 重开工程
- 在 **Bend** 与 **Formant** 上各画明显曲线（多段）→ 保存工程 → 关闭重开 →
  两条分段轨的曲线**原样恢复**（段数、锚点位置/值一致）。

### 12. 孤儿数据保留（声明收缩不裁剪）
- 这是数据层承诺（序列化整存、不因声明收缩裁剪），单测已覆盖
  （`tests/TuneLab.Tests/PiecewiseAutomationSerializationTests.cs`）。
  UI 侧可粗验：画好 Bend 曲线后换一个不声明 Bend 的声库再换回（若有这样的源），曲线仍应保留——
  与连续轨孤儿数据策略一致。

---

## 备注（验证侧重点）

- 编辑交互完全镜像 pitch（pitch 在音符区、分段轨在参数区，值轴不同），故 3~7 的手感应与画 pitch 一致。
- 分段轨**无默认基线**：锚点值即绝对值，段间取值为 NaN（断开），区别于连续轨的"值 + 默认基线"。
- 参数区的**锚点数值输入小框**是连续轨特性（pitch 无此物），分段轨当前不带——非缺陷，按"参考 pitch"范围。
- 本次不含引擎消费：编辑分段轨不改音频（见准备区"重要范围说明"）。

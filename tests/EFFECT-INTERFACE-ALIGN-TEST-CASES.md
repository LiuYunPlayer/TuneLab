# Effect 接口对齐 + 按段细粒度增量 测试用例

> 本文只覆盖**本次改动的受影响范围**：effect 接口对齐 voice 惯例（持久处理器面）+ effect 链按段细粒度增量过链。
> effect 的基础功能（加载/发现/参数/bypass/删除/链/重排/自动化的功能正确性）见 [EFFECT-TEST-CASES.md](EFFECT-TEST-CASES.md)，不在此重复。
> 本文重点验证两件事：① 接口改造后**行为保持**（基础功能不回归）；② 按段增量缓存**在各种编辑下输出始终正确**（缓存/级联/失效无 stale）。
> 每个用例：**做什么 → 预期看到什么**。

## 准备

```bash
dotnet build TuneLab.sln -c Debug
dotnet build tests/TestPlugins.slnx -c Debug
powershell -File tests/pack-tlx.ps1          # 打成 tests/tlx/*.tlx
```

- 安装：把 `tests/tlx/v1-effect.tlx` 和一个声库包（如 `tests/tlx/v1-voice.tlx` 或 `tests/tlx/v1-suite.tlx`）拖进 TuneLab 窗口（或扩展侧边栏 → Install Extension）。
- 测试 effect 引擎（UI 显示名）：**TLTestGain**（参数 `gain` 0~2 默认 1；自动化轨 **Gain Env** 默认 1；输出 = 输入 × gain × env(t)）与 **TLTestReverse**（无参数，倒放样本）。
- 操作入口：选一条 **MIDI part**，给它选好声库、写若干 note 并能正常合成出声（作为基线），右侧 **Properties** 侧栏的 **Effects** 面板加 effect。
- 让段化可观察：写**两簇分开的 note**（中间留明显停顿），使 voice 产出**多个音频段**（段边界落停顿处）。

## 一、接口对齐后行为保持（不回归）

> 这些是基础功能的快速回归：改造后应与改动前**完全一致**。

### 1. 加载与发现
- 装 `v1-effect.tlx` → 扩展侧边栏 **Loaded**、类别 **effect**；Add Effect 菜单同时列出 **TLTestGain** 与 **TLTestReverse**。

### 2. 单 effect 参数生效
- 加 **TLTestGain**：gain=1 听感不变；拖到 **0** 静音；拖到 **2** 明显更响。改 gain 后**自动重渲染**，无需额外操作。

### 3. bypass / 删除
- 取消勾选启用 → 旁路（回到加 effect 前）；重新勾选 → 恢复。点 ✕ → 移除、音频回原始。

### 4. 链串联 + 重排
- 加 **TLTestGain**(gain=0.5) + **TLTestReverse** → 既变小声又倒放；用 ↑/↓ 重排不报错、按新顺序重渲染。

### 5. 自动化 Gain Env
- 给 Gain 点亮 **Gain Env** 轨并编辑曲线（如前段画到 0、后段画到 2）→ 音频按曲线随时间增益变化（前段静音、后段更响）。

✅ 通过标准：以上全部与改动前一致（接口从「一次性任务」改为「持久处理器」对用户不可见）。

## 二、按段细粒度增量：编辑下输出始终正确

> 段化后每段独立过链、增量重渲染。下列编辑若缓存/级联/失效有 bug 会产出 **stale 或错误音频**；预期始终正确。

### 6. 单段自动化编辑不污染其他段
- 两簇 note（段 A、段 B）+ Gain + Gain Env。先全曲合成出声作基线。
- **只在段 A 的时间范围内**改 Gain Env（如段 A 拉到 0）→ 段 A 变化（静音），**段 B 保持不变**（不被误改、不出现 stale）。
- 反过来只改段 B 范围 → 段 B 变、段 A 不变。

### 7. 只改静态参数 gain（不动 Env）
- 已点 Gain Env 曲线后，单独把 gain 从 1 改到 0.5 → 整体音频在原 Env 形状上整体 ×0.5（Env 曲线形状保留、不丢失、不被重置）。

### 8. 改 Env 不影响 gain
- 设 gain=0.5，再编辑 Gain Env 曲线 → 输出 = 输入 × 0.5 × env(t)（gain 标量仍是 0.5，不被 Env 编辑覆盖）。

### 9. 链中上游变化级联到下游
- 链 **Gain → Reverse**。改 Gain 的 gain/Env → 下游 Reverse 的输出随之更新（倒放的是更新后的 Gain 输出，不是 stale）。

### 10. voice 改动触发段重渲染
- 改某簇 note（音高/歌词/增删）使该段 voice 音频变 → 该段 effect 链用**新音频**重过；其余未变段保持（听感与波形仅变化段更新）。

### 11. bypass 来回切 + 再编辑
- Gain 加 Env 曲线后：取消启用（passthrough，回原始）→ 重新启用 → 效果**完整恢复**（gain + Env 都在，无半状态/无 stale）。
- 连续多次 toggle 不残留、不错乱。

### 12. 链结构变化（增删/重排）后正确
- 播放/合成完成后：插入一个 effect 到链中间、删除、上下移 → 每次都从结构变化处正确重渲染，最终音频与「按最终链顺序一次性过链」一致。

### 13. 采样率切换
- 设置里切换工程采样率 → 各段按新率重做、全链重渲染，音频正确（无错位/无 stale buffer）。

### 14. 编辑中频繁改动不崩、能收敛
- 拖动 Gain Env / 反复拖 gain 滑块期间，UI 不卡死、不报错；松手后音频**收敛到正确结果**（中途取消的重渲染不残留错误产物）。

### 15. 稀疏 part 段间留白
- 两簇 note 中间的停顿区 → 播放/波形该区**留白静音**（段间空洞不画、不补零），与基线一致。

## 备注（验证侧重点）

- 用例 6/9/10 是按段二维失效（段 × 级）的核心正确性：单段编辑不应触发其他段重算、上游变化必须级联到下游、voice 段变必须只重渲该段。**观察点是「输出正确且无 stale」**——增量带来的性能收益（跳过未变段/未变级）对快速的测试 effect 不易从延迟上察觉，但若增量逻辑有 bug 会直接表现为**音频错误**。
- 用例 7/8 验证引擎私有差分复用的正确性：参数与自动化分别失效不同内部内容，互不覆盖。
- 用例 11/12/13 验证处理器生命周期（持久/弃置/重建）在 bypass、结构变、采样率变下无残留。

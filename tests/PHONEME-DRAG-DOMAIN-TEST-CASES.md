# 音素拖拽「统一域模型 v4：刚性平移、弹性吸收、如实入账」 — 手测用例

只覆盖本次需求受影响范围：**拖分界线的消解分配 / 平移语义 / 钳制 / 钉死 / 标称有界性**。
显示生命周期见 `PHONEME-DISPLAY-TEST-CASES.md`；本文档取代旧域模型（相①/相②/Case 2 两段式）的手测口径。

模型要点（供判定期望时对照）：
- **作用域** = 线后音素**归属** note 的域 + 前域（归属看列表成员、不看时间位置），整场拖拽恒定；跨 note 头无感、junction 不是边界（随材料平移）。
- **拖线左移 δ**：线后音素显示 +δ、其后一切冻结；线前刚性（w=0）**整体平移**（长度不变）；线前作用域内弹性核（w>0）按 ∝(w·显示长) 分担吸收（触底出局、水填），**弹性全触底才**轮到刚性均分收缩。右移镜像。
- **如实入账**：只有真实变了显示长的音素才被写标称（平移的不写）；平移越过 junction 的量写进 BodyOffset。**任何拖拽后标称都是物理量级**（不再出现夸张大标称）。
- **钳制**：右 = 线后音素显示归 0（首音素额外不越 note 头）；左 = 前 note 头（打包极限）或消解容量耗尽。

## 前置准备

1. **拿新二进制**：先**关掉 TuneLab**，再跑 App。
2. 新建空工程 + 一条 MidiPart 挂一个 voice 声库；横向放大到音素带可拖。
3. 观察标称值：侧栏「Note」页 Phoneme 面板逐音素时长框；观察钉死态：右键菜单是否有 "Clear Locked Phonemes"。

## 用例 1 — 双核比例调整（本次动机场景）：线性跟手、标称有界

1. 造一个 note 含两个弹性核（如把某音素权重在侧栏改为 >0，形成 `a1 a2` 双核）。
2. 拖 **a2 的起点线**往左压到很小，再看两核的显示与标称。

**期望**：a1 被压小、a2 占据剩余空间，**线性跟手**；侧栏读数：a1 标称变小、a2 标称变大，**两者都是正常量级**（不再需要把 a2 充到夸张大）；两者之和 ≈ 拖之前之和。来回拖动可完全还原。

## 用例 2 — 刚性平移：拖尾辅音线、辅音不缩短

1. 造 `[k a n]`（k 引导、a 核、n 尾辅音）+ 相接的前 note。
2. 拖 **n 的起点线**左移适中距离。

**期望**：n 变长、n 之后不动；**a 被吸收缩短**；k **平移**（长度不变——侧栏 k 的标称不变）。继续往左拖到 a 触底后，才开始压缩刚性（k 等辅音开始变短）。

## 用例 3 — junction 随动：拖刚性 body 后面的线、junction 被推着走

1. 造 body 首音素为刚性的 note（如全 w=0 的 legacy 类声库，`[X Y]` 均刚性），前 note 相接、有弹性核。
2. 拖 **Y 的起点线**（X 之后那根）左移，越过本 note 头。

**期望**：X **保持长度整体前移**（junction 跟着走、X 跨到拍前），前 note 的核被吸收缩短；线连续跟手、跨头无卡顿；X 的标称不变（平移不入账）。

## 用例 4 — 结合线（核起点）：与普通线同规则

1. `ka` + 相接前 note，拖 **a 的起点线**（结合线）左右往复、并左拖跨头深入前域。

**期望**：k 随 junction 平移（标称不变）、前 note 核被吸收；右拖回来完全可逆；前域塞满时线停住、**侧栏读数无畸大标称**。

## 用例 5 — 钳制与结构极限

1. **右钳**：任一线右拖到底——线停在"线后音素显示归 0"处（贴上下一条线）；**首音素**右拖恰停 note 头、被头切开的拍后半保留、标称不被削没。
2. **左钳**：左拖到底——线最远到前 note 头，且此时线前作用域内的音素显示全被压尽；再拖不动、松手后侧栏无畸形值。
3. **首音素整簇在拍后**（先右拖结合线把引导簇推到头后）：拖首音素起点右移——线不动且**不写数据**（结构平区，松手后标称原样）。
4. **满打包态右拖结合线（2026-07-14 实测回归）**：`s s a` 引导簇左缘恰在 note 头（满打包），拖 **a 的起点线**往右——**期望**：s 簇伸长、a 相应缩短，**显示与标称一致**（junction 实际移多少、a 就少多少）；拖到极限停住后松手，**a 的标称不为 0 除非 a 显示真被压到 0**——绝不出现"线没动、a 标称却被清空"。

## 用例 5b — 前无相接邻居：拍前自由空间（2026-07-14 实测回归）

1. 孤立 note（前面没有相接音符），如 `ki`（k 引导 + i 核 + 尾辅音）。
2. 拖 **k 的起点线**（首音素头）往左拉长：**期望** 自由伸长、1:1 跟手、无左界（不需要有前 note 才能拉）。
3. 拖 **结合线**（i 的起点）往左：**期望** k 整体平移进拍前（k 标称不变），核前伸；再拖回来（右移）：先免费回撤拍前堆叠、k 不被改写；回撤尽（k 左缘贴 note 头）后继续右拖 = k 伸长吃核（与用例 5.4 满打包行为一致）。
4. 往返拖动后检查侧栏：**期望** 数据可逆、无暗账。

## 用例 5c — 全刚性引导 + 核跨拍：引导线双向可拖（2026-07-14 实测回归）

1. 两相接音符如 `ki` + `sa`（s、k 引导 w=0），把 `sa` 的**结合线**左拖使 **a 的左边界跨到拍前**；前 note 的核 i 也设 w=0（或用全 w=0 声库、仅给 a 设 w=1）。
2. 分别拖 **s 的起点线**和 **k 的起点线**，左右往复。

**期望**：左右都能拖。右拖 = 线前引导与线后引导守恒 roll（如拖 k 线右移：s 伸长、k 缩短、a 前半与 junction 冻结不动、线 1:1 跟手）；不再出现"只能往左、不能往右"。

## 用例 5d — 跨拍核右拖：过拍点连续、无提前吸附（2026-07-14 实测回归）

1. 沿用 5c 布局（全刚性 + 仅 a w=1），把 **a 的左边界**拖到拍前较深处。
2. 缓慢**右拖** a 的左边界，穿过拍点继续；观察线前刚性音素在压缩态下逐渐恢复的过程。

**期望**：全程连续跟手——尤其在"线前刚性音素恰好恢复到标称长度"的时刻**不再跳变**（旧 bug：a 的自然起点被前方标称增益提前推过拍点 → 前半提前重分类消失 → 线吸附到拍点）；过拍点前后无缝；来回往复无迟滞。
3. **分类不敏感**：本用例须在两种声源下各测一遍——引导/主体双列表分类的（j、l 在引导列表）与 legacy 全 body 分类的（j、l、a 全在主体列表）。两者行为应一致（跨拍者不必是 body[0]，修复对任意分类成立）。

## 用例 6 — 钉死矩阵

1. 相接三连 A、B、C。拖 B 的**拍后线**（头右侧）：**期望** A、B、C 全被钉死（A=前域参与方、C=本域共享空间）。
2. 拖 B 的**拍前线**：**期望** A、B 钉死，C 不钉。
3. A、B 之间留空隙再拖 B 的线：**期望** A 不被钉死，且左拖 1:1 自由（拍前料自由堆叠、无吸收）。
4. 任一情形 undo 一步：**期望** 拖拽改动连同钉死一起回退。

## 用例 6b — 弹性消解分梯队：junction 不轻动（2026-07-15 裁定）

1. 两相接音符均为纯元音（如 `a` + `a e i`，四个音素全 w=1）。
2. 左拖后 note 的 **a|e 线**或 **e|i 线**。

**期望**：线前的本 note 弹性（a、e）先吸收缩短，**后 note 的 a 的头（junction）保持不动**、前 note 的 a 不受扰；只有近侧弹性拖到触底（1ms 地板）后，junction 才开始被推入前域、前 note 的 a 才开始吸收。右拖镜像：近侧弹性先回胀、junction 不被拉动。近侧无弹性（junction 右全刚性）时行为不变——立即进入越 junction 通道（junction 随材料平移）。

## 用例 7 — 可逆性与数据卫生（回归重点）

1. 任选一条线，左拖一段→右拖回原位→松手。
2. 检查侧栏所有相关音素标称与 BodyOffset。

**期望**：与拖之前一致（同一手势内基线参数化，往返精确还原）；无任何"显示没变但标称被改"的暗账（平区不写数据）。

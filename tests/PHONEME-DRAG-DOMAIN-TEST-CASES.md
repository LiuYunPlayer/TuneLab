# 音素拖拽「统一域模型」 — 手测用例

只覆盖本次需求受影响范围：**拖分界线的域判定 / 范围钳制 / 跨域（Case 2）钉死 / 极限行为**。
不重复已通过的基线（相①弹性吸收 + 相②刚性守恒转移的核心手感见 `PHONEME-EDIT-DIRECTION-TEST-CASES.md`，显示生命周期见 `PHONEME-DISPLAY-TEST-CASES.md`）。

模型要点（供判定期望时对照）：
- **域** = 相邻两个内容 note 头之间的区间（乘客/延音符透明；不相接则各自独立）。**主域** = 拖拽起手时线的显示位所在域（拍前 ⇒ 前域、否则本域）。
- **双向无感跨域**：一次拖拽手势内线可连续跨过 note 头进入相邻域（左跨进前域 / 右跨回本域），无需松手分步。
  唯一硬极限是结构性的：**首音素起点线不越 note 头右侧**（域内材料从头起排），右拖到头即停、拍后半保留。
- **起手钉死主域参与者**：拍前线 ⇒ 本 note + 前内容邻居；拍后线 ⇒ 本 note + **后内容邻居**（其拍前料借入本域共享空间）。有空隙（不相接）则只钉本 note。
- **Case 2 就地钉死**：线实际跨过 note 头的那一刻，目标域的另一参与者才被钉死（左跨 ⇒ 前内容邻居、右跨 ⇒ 后内容邻居）；拖了但没跨，保持合成态。
- 统一旋钮：每根线改**线后音素标称时长**；结合线额外同步位移 BodyOffset。刚性转移仅在域内弹性耗尽后触发。

## 用例 1 — 起手钉死矩阵：拍后线连带钉**后**邻居

1. 造两个**相接**音符 A、B（如 `ka` + `sa`，B 带前置辅音 `s`）。
2. 拖 **A 的拍后线**（如 A 内 a|尾辅音 分界、或 A 核起点右侧任一线），随便动一下后松手。

**期望**：A 与 **B** 均被钉死（B 的拍前料与 A 共享域空间，一并冻结）；A 的**前**邻居（若有）不被钉死。undo 一步应连钉死一起回退。

## 用例 1b — 拍前线连带钉**前**邻居（回归）

1. 同上布局，拖 **B 的拍前线**（`s` 起边界，显示在 B 头之前）。

**期望**：B 与 **A** 被钉死；B 的后邻居不被钉死。

## 用例 1c — 不相接不钉邻居

1. 把 A、B 拉开一段空隙，拖 B 的拍前线 / A 的拍后线。

**期望**：只有被拖的 note 自己被钉死，邻居保持合成态；拍前线拖动 1:1 跟手（自由堆叠、无吸收）。

## 用例 2 — 拍前线右跨回本域：一次手势无感穿越

1. 造 B 带**两个**前置辅音（如 `sta`），整簇显示在 B 头之前，且**结合线在 B 头之后**（核起点晚于头，头与结合线之间有空间）。
2. 把 **t 的起边界**（第二根拍前线）往右**一口气拖过** B 的 note 头，不松手。

**期望**：线**连续穿过 note 头**进入本域，不卡顿、不用松手分步；穿过后 `s` 跨头（其拍后半填在头右侧），本域核吸收、前域（A 的核）回涨，全程跟手无跳变。

## 用例 2b — 首音素线右拖的结构性极限

1. 同上布局，把**首音素线**（`s` 起边界）往右猛拖、试图越过 note 头。

**期望**：线**恰停在 note 头**（首音素起点结构上不可能显示在头右侧——域内材料从头起排），被头切开的拍后半保留、`s` 标称不被削没；往回拖立即跟手。

## 用例 3 — 首音素整簇落拍后时右拖不锁死（bug 修复）

1. 造 `sta`，先把**结合线**（核 `a` 起点）右拖，使 `s`、`t` 整簇都显示在 note 头**之后**。
2. 再拖 **s 的起边界**（此时它是本域内的线）往右。

**期望**：能正常右拖（缩短 `s`）——修复前此线右拖完全锁死。

## 用例 4 — Case 2：跨过 note 头才钉目标域参与者（双向）

**左跨**：
1. A、B 相接，B 的引导簇显示在 B 头**之后**（同用例 3 的摆法）。此时拖 B 的引导线起手只钉 B + B 的后邻居（若有）。
2. 把该线往左拖但**不越过** B 的 note 头，松手 → 检查 A：**不应**被钉死。
3. 再次拖同一根线，**越过** B 的 note 头进入 A 的域，松手。

**期望**：越过的那一次 A 被钉死；线进入前域后由 A 的核吸收、继续跟手（一次拖拽内连续无跳变）；undo 连 A 的钉死一起回退。

**右跨**（对称）：
4. B、C 相接（C 在 B 之后、带拍前料），拖 B 的某根**拍前线**（显示在 B 头之前，起手钉 A + B）。
5. 不越过 B 头松手 → C 不被钉死；一口气拖过 B 头再松手 → C 被钉死。

## 用例 5 — 结合线左拖极限：塞满即停、不畸大

1. A、B 相接，A 排满刚性辅音（或用全 `w=0` 的 legacy 类声库），B 为 `ka`。
2. 把 B 的**结合线**（`a` 起点）往左狂拖到底，来回多次。

**期望**：线在**前域塞满处停住**（A 的辅音不被无限压瘪）；松手后侧栏看 `k`/`a` 标称时长为合理值（不畸形巨大）；再往回拖立即跟手（无死区回弹）。

## 用例 6 — 弹性线后：单弹性不动、数据不被污染

1. 造 note 含两个 w>0 音素（侧栏把某音素权重改 >0，如 `kaa` 双核），域内**只留一个弹性**时先试：
2. 拖某**弹性音素的起边界**（非结合线，如第二个 `a` 的起点且线前是刚性时不算——要选"线后是 w>0"的线）：
   - 域内**仅此一个弹性**：线应**基本不动**（合理——要动去拖刚性侧）；松手后侧栏确认其标称**未被改成怪值**（无视觉变化不写数据）。
   - 域内**两个弹性**：拖动有效（份额重分），跟手方向正确、无发散。

## 用例 7 — 回归：核心手感不变

跑一遍 `PHONEME-EDIT-DIRECTION-TEST-CASES.md` 用例 4/4b/5（全刚性 roll、弹性吸收 1:1、耗尽转刚性转移不畸大）。

**期望**：全部照旧通过。

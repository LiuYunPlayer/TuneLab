# Note 重叠（和弦）创建/编辑入口 — 手测用例

只覆盖本需求受影响范围：**面向用户的重叠 note 创建/编辑入口**。上一需求已打通数据层与合成直传（见
`VOICE-NOTE-OVERLAP-TEST-CASES.md`，那份验证"已有重叠 note 经插件消费"的听感，不在此重复）。

本需求三条入口/出口：

1. **钢琴窗自由重叠** —— 拉伸 note 不再自动截断/删除被盖到的邻居，可直接画出重叠/和弦。去重叠责任在插件（SDK 契约本就直传可重叠序列、下放插件消费），编辑器不再代为规避。
2. **MIDI 导入保真** —— 不再把每个 note 尾巴钳到下一 note 起点，重叠/和弦原样保留。
3. **一键去除重叠命令** —— 钢琴窗（选中 note）与轨道窗（选中 part）右键菜单，镜像合成侧"后盖前"把重叠整理成单声部，裁到 0 的 note 删除。

> 没有引入任何声源能力位或编辑器模式开关：重叠创建对所有声源一致放开；想单声部的用户用入口 3 整理，或交由插件/compat 在合成侧消费。

## 前置准备

1. **构建并重打测试插件包**：
   ```
   dotnet build tests/TestPlugins.slnx -c Debug
   pwsh tests/pack-tlx.ps1
   ```
2. **安装测试声库**（拖进窗口或扩展侧边栏 Install Extension）：
   - `tests/tlx/v1-voice.tlx` → V1 会话声库，UI 显示名 **「Alice (V1 Test)」**（原样消费可重叠 note，能把和弦发声，用于听端到端叠音）。
   - `tests/tlx/legacy-voice.tlx` → 老接口声库，UI 显示名 **「Carol (Legacy Test)」**（经 compat「后盖前」钳成单声部，作对照）。
3. 准备一个空工程，新建一条 MidiPart 备用。

## 用例 1 — 钢琴窗自由画出重叠/和弦

挂 **「Alice (V1 Test)」** 声库。在钢琴窗画两个相邻 note：A 在前、B 紧随其后。

1. **拉伸成尾部重叠**：拖 A 的**右边界**向右越过 B 的起点。
   - **期望**：A 变长、与 B 重叠，**B 不被裁剪也不消失**（旧行为会裁/删 B）。
2. **反向拉伸**：拖 B 的**左边界**向左越过 A 的终点。
   - **期望**：同样形成重叠，A 不变。
3. **叠成和弦**：在某个 note 上方/下方同位置双击创建第二个 note。
   - **期望**：同起点同时长的多个 note 并存。
4. **播放**：Alice 声库可听到重叠区叠音 / 同起点和弦（V1 引擎原样消费）。挂 Carol 同一工程播放则经 compat 钳成单声部听感（数据仍重叠，仅合成侧兜底）。

> 判定：拉伸/创建永远不再吞掉邻居；overlap 由数据保留，听感差异来自下游插件/compat，而非编辑器。

## 用例 2 — MIDI 和弦导入保真

夹具：`tests/fixtures/midi-chord-import.mid`（480 PPQ，与 voice-note-overlap.tlp 同布局）：

| note | pos (tick) | dur (tick) | pitch | 用途 |
|---|---|---|---|---|
| 1 | 0 | 720 | C4(60) | 尾部重叠：被 note2 盖 |
| 2 | 480 | 720 | E4(64) | 起点落在 note1 内 |
| 3 | 1920 | 720 | C4(60) | 同起点和弦：最长 |
| 4 | 1920 | 480 | E4(64) | 同起点和弦：中 |
| 5 | 1920 | 240 | G4(67) | 同起点和弦：最短 |
| 6 | 2880 | 480 | D4(62) | 和弦后的独立 note |

步骤：菜单导入 / 拖入该 `.mid`。

- **期望（保真）**：导入后 **6 个 note 全部存在**，note1 与 note2 在 [480,720) 重叠，note3/4/5 同起点 1920 叠成和弦。
- **对照（旧钳制行为，不应再出现）**：note1 被裁到 480；和弦只剩最短的 G4（note3/4 钳后归零被删）——若看到这个结果说明改动失效。

挂 **「Alice (V1 Test)」** 播放可听到叠音 + C 大三和弦（验证端到端）。

## 用例 3 — 一键去除重叠（钢琴窗 + 轨道窗）

去重叠算法镜像合成侧"后盖前"单声部钳位：按数据序（StartPos 升 → EndPos 降）把每个 note 尾巴钳到
下一 note 起点；钳后时长 ≤ 0 者（被完全覆盖的前者、同起点的较长和弦兄弟）删除。

先用用例 2 导入的重叠工程，或用例 1 手画的和弦作为输入。

### 3a 钢琴窗 · 作用于选中 note
1. 框选含重叠的若干 note（如 note1+note2，或整段和弦）。
2. 右键 → **「去除重叠 / Remove Overlaps」**。
3. **期望**：
   - note1 尾巴钳到 480 → C4 [0–480)，E4 [480–1200)，无重叠区。
   - 同起点和弦 → 只剩最短的 **G4 [1920–2160)**，C4/E4 删除。
   - 仅整理选中集合：未选中的 note 不受影响。
4. 一次 Undo 应整体还原。无重叠时点击应**无操作、不产生 Undo 步**。

### 3b 轨道窗 · 作用于选中 part（支持多选）
1. 回轨道窗，框选/全选含重叠的一个或多个 MidiPart。
2. 右键 → **「去除重叠 / Remove Overlaps」**。
3. **期望**：每个选中 part 整体整理为单声部（同 3a 的"后盖前"结果），多选 part 一次完成；一次 Undo 整体还原。

## 受影响范围之外（本次不验证）

- 已有重叠 note 经 voice/legacy 插件消费的听感与产物归属 → 见 `VOICE-NOTE-OVERLAP-TEST-CASES.md`。
- 歌词连音 `-` / phoneme 越界压缩比：维持现状（重叠场景由插件/compat 在合成侧处理）。
- 钢琴窗重叠 note 的命中/分层显示、缝隙拖拽 seam 语义：后续交互/视觉优化话题。

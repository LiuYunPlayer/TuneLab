# 音素编辑「生长方向 + 全刚性 roll」 — 手测用例

只覆盖本次需求受影响范围：**拖音素分界线 / 侧栏改音素时长时的生长方向**，以及**全刚性域拖分界线的 roll（不发散、自然长有界）**。
不重复已通过的基线（钉死时长模型见 `PHONEME-ANCHOR-TEST-CASES.md`、显示生命周期见 `PHONEME-DISPLAY-TEST-CASES.md`、侧栏面板结构见 `PHONEME-EDITOR-TEST-CASES.md`）。

> 模型升级（2026-07：`Preutterance` 标量 → `LeadingPhonemes`/`BodyPhonemes` 双列表 + 有符号 `BodyOffset`）后**可观测行为不变**，
> 仅内部机制变了：生长方向不再靠「拍前占比 `f` 耦合 Preutterance」派生，而是**由音素所属列表直接决定**——改**引导**音素 → 向左长（`BodyOffset` 不变、主体固定）、改**主体**音素 → 向右长。
> 下方各用例的期望（向左/向右/跨拍分部、note 头不动、roll 守恒）全部照旧成立；文中 `Preutterance` 与 `f` 是等价机制的历史表述（宿主拖拽反解内部仍用等价参数 `P = Σleading − BodyOffset` 复用旧反解数学）。

模型要点（供判定期望时对照）：
- **生长方向由所属列表决定**：改**引导**音素 → 向左长（前置辅音，右缘=结合线不动，主体整体固定）；改**主体**音素 → 向右长。等价旧述：占比 `f` 纯拍前=1、纯拍后=0、跨拍按比例，改长 δ 时 `Preutterance += f·δ`（= `BodyOffset` 不变）。
- **nominal 神圣**：编辑只改被操作音素自己的时长（引导编辑必要时随动其 note 的 `BodyOffset` 保主体不动），不改邻居核的 nominal。
- **有弹性核（w>0）**：改刚性音素长度 → 核自动吸收，其余音素不动、1:1 跟手。
- **全刚性域（无核，legacy 全 `w=0`）**：拖分界线走 **roll**——两侧相邻音素转移自然长（守恒和），边界线性跟手、域端点固定、自然长有界（不发散）。跨 note（拖首音素、相接前邻）时转移对象是**前邻末音素**，前邻会被一并钉死。

## 前置准备

1. **拿新二进制**：先**关掉 TuneLab**，再跑 App（自动 build 最新宿主）。
2. 新建空工程 + 一条 MidiPart 挂一个 voice 声库。
3. **横向放大**到音素带（钢琴窗下方波形区）可见、可拖分界线。
4. 在音素带上**拖某 note 的任一音素起边界**即把该 note 锁为钉死；侧栏「Note」页的 Phoneme 面板逐音素列出「时长 / 权重」数值框。

## 用例 1 — 侧栏改前置辅音时长：向左长（拍前）

1. 造一个带前置辅音的音节（如 `ka`：`k` 前置 `w=0`、`a` 核 `w=1`），先拖一下锁定。
2. 侧栏找到 `k` 的时长框，**增大**其值。

**期望**：`k` 的**左缘向左伸**（越过 note 头、往更早的时间延伸），**右缘（k|a 分界）不动**；`a` 不受影响。减小则 `k` 左缘右收。（即 `Preutterance` 随 `k` 时长同增同减，`f=1`。）

## 用例 2 — 侧栏改拍后音素时长：向右长（拍后）

1. 同上音节，侧栏改 `a`（核，拍后）时长——或造带后辅音的 `kas`，改 `s`（后辅音，拍后）时长。
2. 增大后辅音 `s` 的时长。

**期望**：拍后音素**右缘向右伸**、**左缘不动**（`Preutterance` 不变，`f=0`）。核 `a` 时长框改动对显示可能几乎无变（单核填充，nominal 是软比例）——这是既有行为，非本用例重点。

## 用例 3 — 侧栏改跨拍音素时长：按比例分

1. 造一个 note 头切在某音素内部的跨拍情形（半元音 / 首音素跨 note 头）。
2. 改该跨拍音素时长。

**期望**：左缘与右缘**按拍前占比 `f`** 各分一部分位移（左缘移 `−f·δ`、右缘移 `+(1−f)·δ`），note 头恒不动，跨拍占比保持。无跳变。

## 用例 4 — 全刚性 · 跨 note 拖分界线（本次核心 bug）

1. 用**全 `w=0`**（legacy 类，所有音素刚性）声库，造两个相接音符，如 `sa`(s,a) + `ka`(k,a)，`a` 与 `k` 均 `w=0`。
2. 横向放大，拖 **a|k 分界线**（`ka` 的 `k` 起边界），来回拖、拖到快挤没前元音 `a`。

**期望**：
- 分界线**跟手、不跳**（鼠标一动不再瞬间窜到后面）。
- 越拖越挤 `a` 时**灵敏度正常、不发散**（不需要越来越大的位移）。
- 松手位置 = 拖停位置。
- **关键验证自然长有界**：把两音符接缝**稍微拉开**（断开相接）后再看 `k` 的实际长度——应是**合理值**（≤ `a+k` 的原自然和），**不是**畸形巨长。
- `ka` 的核 `a`（`k` 之后）与 `sa` 的 `s` 不被拖乱（各自长度不变、仅平移）。

## 用例 4b — 全刚性 · 前音符元音开头在拍前，拖后音符辅音（回归）

1. 全 `w=0` 声库。造 A、B 两相接音符，让 **A 的元音开头落在拍前**（元音跨 A 的音符头：一部分在 A 头之前、一部分在 A 内）。B 有前置辅音。
2. 拖 B 的前置辅音起边界（= A 元音与 B 辅音之间那条线），来回拖。

**期望**：
- B 的辅音伸缩、A 元音的**拍后侧（A 头之后那段）**随之让位——**A 元音的拍前部分（A 头之前那段）纹丝不动**。
- A 的音符头切分（Preutterance）不被改动；A 头之前的音素/几何完全不受影响。
- 曾经的 bug：拖 B 辅音会连带把 A 的拍前音素一起拖动——现应消失。

## 用例 5 — 全刚性 · 同 note 内拖分界线

1. 全 `w=0` 声库，造一个多刚性音素的 note（如后辅音簇 `kalt`：`l`,`t` 均拍后 `w=0`）。
2. 拖 `l|t` 分界线。

**期望**：`l` 与 `t` 之间转移长度（一长一短、守恒和），**其余音素与 note 末端不动**，边界线性跟手、有界。

## 用例 6 — 非回归 · 有核域拖分界线仍 1:1

1. 普通声库（元音 `w>0`），造 `ka`，拖 `k` 的起边界。

**期望**：`k` 边界**1:1 跟手**（无压缩发散），元音 `a` 吸收变化、其余音素与核的记录不被篡改。行为与本次改动前一致（本次不改有核路径）。

## 用例 7 — 撤销 / 提交为一步

1. 上述任一编辑后按 **Ctrl+Z**。

**期望**：一次编辑（拖一程 / 改一次数值框）= **一个撤销步**，整体回退（含被 roll 连带钉死 / 修改的前邻）。
